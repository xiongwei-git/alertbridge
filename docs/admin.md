# 管理后台指南

管理后台位于 `/admin/`。生产环境必须通过宝塔配置的 HTTPS 域名访问，容器端口只绑定宿主机回环地址。

## 登录和密钥

- 默认引导用户名为 `admin`，部署时可通过 `ALERTBRIDGE_ADMIN_USERNAME` 修改；
- 管理员密码由部署者提供，首次启动后只保存 Argon2id 哈希；
- 管理员已经存在时，重启或修改 Compose 引导值不会覆盖登录凭据；
- 动态配置主密钥由服务自动生成到独立命名卷，不显示、不写日志；
- 浏览器会话默认 12 小时，Cookie 使用 `Secure`、`HttpOnly` 和 `SameSite=Strict`；
- 新建或轮换 HMAC 客户端、轻量接入令牌时，凭据只显示一次，丢失后只能重新轮换。

`alertbridge-data` 和 `alertbridge-secrets` 必须作为一组备份。主密钥丢失后，SQLite 中的客户端和渠道密文无法恢复。

## 空系统配置顺序

新部署不会预置示例客户端、默认路由或通知渠道：

1. “通知渠道”：创建渠道并发送测试消息；
2. “路由规则”：用 `routing_key + severity` 绑定一个或多个渠道；
3. “客户端”：选择现有路由，按需求创建轻量令牌或 HMAC 调用身份并保存一次性凭据；
4. “接入指南”：复制已经代入当前域名、客户端和可用路由的示例。

后台修改写入 SQLite 后立即生成新的只读运行快照，无需重启容器。删除渠道后，尚未发送的对应任务会进入死信，避免静默丢失。

## 页面功能

- 运行概览：尚未恢复的活跃故障、今日事件、排队、重试、死信和成功投递统计；
- 客户端：管理 HMAC 客户端及其路由授权，也可创建固定路由、级别和独立限额的轻量令牌；
- 通知渠道：飞书、Telegram、ntfy、SMTP，支持启停和测试发送；
- 路由规则：按路由和严重程度选择一个或多个已配置渠道；
- 静默窗口：限定路由、严重程度和最长 30 天的时间范围；
- 投递记录：筛选投递状态、查看安全化错误并重新入队死信；
- 接入指南：生成轻量通知配置，以及完整事件格式、签名规则和 Shell/Python 示例。

技术名词旁的 `?` 支持鼠标悬停、键盘聚焦和触屏查看，不依赖 JavaScript 或外部资源。

## 事件状态与活跃故障

“活跃故障”不是投递失败数量，而是状态仍为 `firing`、尚未收到匹配 `resolved` 的故障数量。故障由 `客户端 ID + routing_key + dedupe_key` 唯一标识：

- `firing`：创建或刷新一个活跃故障；去重窗口内的重复事件仍会记录，但不会重复投递；
- `resolved`：关闭同一故障并计算持续时间；找不到对应 `firing` 时作为孤立恢复事件抑制；
- `info`：普通一次性通知，不进入活跃故障统计；
- `test`：接入或渠道测试，不进入活跃故障统计。

测试调用应使用 `status=test`。真正恢复故障时，`resolved` 必须沿用原 `firing` 的客户端、`routing_key` 和 `dedupe_key`。渠道投递失败请在“投递记录”的 `retrying` 或 `dead` 状态中检查。

## 渠道约束

- 飞书：生产只接受 `open.feishu.cn` 或 `open.larksuite.com` HTTPS Webhook。若机器人启用自定义关键词，后台必须填写匹配的任一关键词，发送时会自动加入正文或卡片标题；
- Telegram：生产固定使用官方 `https://api.telegram.org`；
- ntfy：填写完整 HTTPS topic URL，Token 可选；
- SMTP：只允许隐式 TLS 或 STARTTLS，不支持明文 SMTP。

敏感字段更新时留空表示保留原值；飞书关键词不是密钥，留空会清除。后台不会回显已保存的渠道密码和 Token，也不会把它们写入日志。

## 客户端接入

接入指南根据严格校验后的请求 Host，以及宝塔传递的 `X-Forwarded-Proto`，生成当前 HTTPS 地址。它不会回显已经保存的 HMAC 密钥或轻量令牌。

- `POST /api/v1/notifications`：适合宝塔自定义消息通道和简单脚本。创建令牌时必须选择一条现有的 `routing_key + severity` 规则，并设置 1–60 次/分钟限额；来源、路由、级别由令牌固定，状态固定为 `info`。
- `POST /api/v1/events`：适合需要幂等、防重放、调用方选择路由和 `firing/resolved` 故障生命周期的客户端，必须携带四个 HMAC 请求头。

轻量令牌只在创建或轮换时显示一次，数据库只保存令牌摘要。停用会立即拒绝新请求，轮换会立即废止旧令牌。不要把令牌放入 URL；宝塔应把它写在固定的 `Authorization: Bearer ...` 请求头中。

## 安全运维

应用已有持久化登录限流、串行化内存硬密码校验、CSRF、防点击劫持、严格 CSP 和无缓存响应。仍建议在宝塔 Nginx 对 `/admin/login` 增加来源限速。

登录失败时依次检查：使用的是否为首次初始化的账号密码、HTTPS 是否生效、Nginx 是否传递正确协议，以及浏览器是否拒绝 Secure Cookie。直接修改 `.env` 不会重置已存在的管理员密码。
