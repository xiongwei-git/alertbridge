# 管理后台指南

管理后台位于 `/admin/`，必须通过宝塔配置的 HTTPS 域名访问。容器端口只绑定本机，不要绕过反向代理把后台直接暴露到公网。

## 登录和密钥

- 默认用户名：`admin`；可在首次启动前修改 `config/config.json`。
- 密码文件：`secrets/admin-password`。初始化脚本生成高熵随机值。
- 主密钥：`secrets/master-key`。它用于 AES-256-GCM 加密数据库中的客户端密钥和渠道配置，不能作为登录密码使用。
- 浏览器会话默认 12 小时，Cookie 使用 `Secure`、`HttpOnly` 和 `SameSite=Strict`。
- 新建或轮换客户端时，HMAC 密钥只显示一次。丢失后只能重新轮换。

`master-key` 丢失后，数据库里的动态配置无法恢复。备份数据卷时必须同时将它保存到加密的异地位置。

## 配置生效规则

首次启用后台时，服务把 `config/config.json` 中的客户端、飞书渠道和路由写入数据库，敏感字段立即加密。数据库中已有客户端后，JSON 仅保留为启动所需的初始配置，不再覆盖后台修改。

后台修改成功后会生成新的只读运行快照并立即生效，无需重启容器。正在处理的投递会继续使用当前渠道 ID；删除渠道后，尚未发送的对应任务会进入死信，避免静默丢失。

## 页面功能

- 运行概览：活跃故障、今日事件、排队、重试、死信和成功投递统计。
- 客户端：创建、启停、限定逻辑路由、分钟限流、轮换密钥和删除。允许路由从现有路由名称中直接勾选，不需要手工输入；编辑时会保留客户端已经引用的名称。
- 通知渠道：飞书、Telegram、ntfy、SMTP 邮件；支持启停和发送测试通知。
- 路由规则：按 `routing_key + severity` 绑定一个或多个渠道，目标渠道从已配置渠道中勾选。
- 静默窗口：从现有路由中选择作用范围，并限定严重程度和最长 30 天的时间范围。事件仍记录，但不生成投递。
- 投递记录：按状态筛选，查看安全化的错误摘要，并将死信重新入队。
- 接入指南：选择现有客户端，查看统一事件格式、签名规则，以及自动代入可用路由的 Shell/Python 示例。

页面中容易混淆的技术名词旁有 `?` 说明按钮。鼠标悬停、键盘聚焦或触屏点击该按钮都可查看解释，不依赖 JavaScript 或外部资源。

## 客户端接入流程

1. 在“客户端”创建调用身份，立即复制并安全保存只显示一次的密钥；
2. 从密钥页面进入“接入指南”，或随时从左侧导航打开；
3. 选择客户端后，页面会从它的允许路由和当前路由矩阵中选出实际可用的路由与严重程度，并代入 Shell 和 Python 示例；
4. 页面会根据当前访问地址自动代入 `BASE_URL`；通过宝塔域名访问时会读取 Nginx 设置的 `X-Forwarded-Proto` 并生成 HTTPS 地址，只需粘贴刚保存的密钥；
5. 收到 `202 Accepted` 后，到“投递记录”确认外部渠道最终发送结果。

接入指南只读取非敏感的客户端 ID、允许路由和路由矩阵，不会解密或回显客户端密钥。`POST /api/v1/events` 是唯一正式入站接口，调用方必须提交统一事件正文并携带四个 HMAC 请求头；如果原始监控工具不能做到，应在来源侧或中间增加可信的适配器。

## 渠道约束

- 飞书：生产环境只接受 `open.feishu.cn` 或 `open.larksuite.com` HTTPS Webhook。若机器人启用了“自定义关键词”，请在渠道中填写匹配的任一关键词；AlertBridge 会自动将它加入文本正文或卡片标题。飞书允许配置多个关键词，只要消息命中其中一个即可，详见[飞书自定义机器人使用指南](https://open.feishu.cn/document/ukTMukTMukTM/ucTM5YjL3ETO24yNxkjN)。
- Telegram：生产环境固定使用官方 `https://api.telegram.org`。
- ntfy：必须填写完整 HTTPS topic URL；Token 可选。
- SMTP：只允许隐式 TLS 或 STARTTLS，不支持明文 SMTP。

渠道密码和 Token 留空表示更新时保留原值；飞书关键词不是密钥，留空会清除该配置。后台不会回显已保存的敏感字段，也不会把它们写入日志。

## 安全运维

建议在宝塔 Nginx 对 `/admin/login` 再增加按来源 IP 的速率限制。应用自身已有持久化登录限流、CSRF、防点击劫持、严格 CSP 和无缓存响应，但反向代理层限流可以更早拦截公网扫描。

后台返回登录页、健康检查正常但登录失败时，依次检查：HTTPS 是否生效、系统时间是否正确、`admin-password` 是否带了多余空格，以及浏览器是否拒绝了 Secure Cookie。生产环境不要把 `secure_cookie` 改为 `false`。
